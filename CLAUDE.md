# website — Zaverose art portfolio

Static **Astro 5 + React islands** site. A brutalist "technical software"
portfolio + commission storefront for the furry artist **Zaverose**, replacing
the old carrd export in `../carrd/` (archived, don't touch it).

Read `PRODUCT.md` for who/what/why and `DESIGN.md` for the visual system before
making changes. This is an independent git repo inside the `caravan-app`
meta-repo (each subdir is its own world — see the meta-repo `CLAUDE.md`).

## Stack

- Astro 5, `@astrojs/react` islands, TypeScript **strict**.
- Tailwind v4 via `@tailwindcss/vite` — CSS-first `@theme` in
  `src/styles/global.css` mapping onto runtime CSS variables in
  `src/styles/tokens.css`. **Tokens are the single source of truth for design.**
- Astro Content Collections with the `file()` loader → one JSON manifest.
- React Three Fiber for the landing hero; hls.js for the STREAM player.
- Static output. No server runtime.

## Scripts

```
npm run check      # astro check (types)
npm run import:e621  # regenerate src/content/art/zaverose.json from e621 API
```

```

  content/art/zaverose.json  artwork manifest (generated by import:e621)
  data/       config.ts (mode/rating/brand/cookies), socials.ts, commission.ts
  lib/        mode.ts, rating.ts, tags.ts, pricing.ts, clipboard.ts, art.ts
  layouts/    BaseLayout.astro
  components/ Sidebar, Wordmark, Nav, PageHead (static Astro shell)
  islands/    ModeControl, Settings, HeroCanvas, Slideshow, LinksHub,
              StreamVault, VideoPlayer; gallery/*; commission/*
  pages/      index, art/, art-full.json.ts, links, stream, commission
  styles/     tokens.css (design vars), global.css (@theme + utilities)

```

`@/` is an alias for `src/`.

## Core mechanic — SFW / NSFW mode

- Driven by `<html data-mode="sfw|nsfw">` + cookies (`src/lib/mode.ts`,
  `src/data/config.ts`). Rating logic is `src/lib/rating.ts` (e621 `s`/`q`/`e`).
- **SFW is the SSR default** so explicit URLs never appear in prerendered HTML.
  The explicit set is lazy-fetched client-side from `/art-full.json` only after
  the visitor age-verifies into NSFW mode.
- First NSFW switch → 18+ age gate (cookie-remembered). NSFW flips the wordmark
  to HATE THIS ARTIST and the accent to red.
- The `Astro.request.headers` build warnings are expected: pages are prerendered
  (default SFW), and a pre-paint inline script in `BaseLayout` applies the real
  mode on the client.

## Conventions

- **Islands own their styles**: markup + a trailing `<style>{`…`}</style>` block
  that references the CSS variables. `src/islands/ModeControl.tsx` is the
  canonical reference — follow it for new islands.
- **Don't hardcode colors/spacing** — use the tokens. The accent is the only
  color that changes with mode.
- Follow the impeccable anti-slop rules in `DESIGN.md` (no gradient text, no
  ghost cards, no side-stripe borders, no over-rounding, no image-on-hover).
- Artwork images are e621 CDN URLs (`thumb` = sample, `full` = hi-res). The
  schema fields are just URLs, so self-hosting later needs no code changes.
- Verify with `npm run check` **and** `npm run build` before calling work done.
